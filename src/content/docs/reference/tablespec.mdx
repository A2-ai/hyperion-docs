## Description

Table specification for parameter tables

## Usage

```r
TableSpec(
  title = "Model Parameters",
  name_source = "name",
  columns = NULL,
  add_columns = NULL,
  drop_columns = NULL,
  hide_empty_columns = TRUE,
  sections = section_rules(),
  row_filter = filter_rules(),
  display_transforms = list(),
  n_sigfig = 3,
  ci_level = 0.95,
  n_decimals_ofv = 3,
  pvalue_scientific = FALSE,
  pvalue_threshold = NULL,
  footnote_order = c("summary_info", "equations", "abbreviations")
)
```

<td><code>name_source</code></td>
<td>Which name field to use from ModelComments: "name" (default), "display", or "nonmem_name". Controls how parameter names appear in the output table. Use "nonmem_name" to show raw NONMEM names like "THETA1", "OMEGA(1,1)".</td>

<td><code>columns</code></td>
<td>Character vector of columns to include in output.</td>
</tr>
<tr>

<td>Character vector of columns to append to the column list. Useful for comparisons when you want to add columns like "pct_change" without overriding <code>columns</code>.</td>
</tr>
<tr>
<td><code>drop_columns</code></td>
<td>Character vector of columns to exclude from output, or NULL (default) to include all columns.</td>
</tr>
<tr>
<td><code>hide_empty_columns</code></td>
<td>Logical. If TRUE, columns that are all NA/empty are automatically hidden unless explicitly requested via <code>columns</code> or <code>add_columns</code>. Default is TRUE.</td>
</tr>
<tr>
<td><code>sections</code></td>
<td>Section rules created with <code>section_rules()</code>.</td>
</tr>

<td><code>row_filter</code></td>
<td>Filter rules created with <code>filter_rules()</code>.</td>
</tr>
<tr>
<td><code>display_transforms</code></td>
<td>Named list specifying which transforms to apply for display. Names are parameter kinds (theta, omega, sigma), values are which columns to transform ("all", "estimate", "cv", "rse", "ci", "symbol").</td>
</tr>
<tr>
<td><code>n_sigfig</code></td>
<td>Number of significant figures for numeric formatting in the output table. Must be a positive integer. Default is 3.</td>
</tr>
<tr>
<td><code>ci_level</code></td>
<td>Confidence interval level, between 0 and 1. Default is 0.95 for 95% confidence intervals.</td>
</tr>
<tr>
<td><code>n_decimals_ofv</code></td>
<td>Number of decimal places for OFV values in summary footnotes. Use NA to keep significant-figure formatting. Default is NA.</td>
</tr>
<tr>
<td><code>pvalue_scientific</code></td>
<td>Logical. If TRUE, p-values are formatted in scientific notation. If FALSE (default), uses significant figures from n_sigfig.</td>
</tr>
<tr>
<td><code>pvalue_threshold</code></td>
<td>Numeric or NULL. If set, p-values below this threshold are displayed as "< threshold" (e.g., "< 0.05"). Default is NULL (no threshold).</td>
</tr>
<tr>
<td><code>footnote_order</code></td>
<td>Character vector controlling the order of footnote sections, or NULL to disable footnotes. Valid values: "summary_info", "equations", "abbreviations". Default is c("summary_info", "equations", "abbreviations").</td>
